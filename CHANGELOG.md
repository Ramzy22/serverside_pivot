# Changelog

All notable changes to `pivot-engine` will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

---

## [Unreleased]

### Added

- Multi-instance safety contract: `session_id`, `client_instance`, and `table`-scoped request isolation
- `SessionRequestGate`: per-instance request deduplication with epoch, sequence, and abort-generation tracking
- `PivotRuntimeService`: unified service layer for server-side data fetching from Dash callbacks
- `register_dash_pivot_transport_callback`: single-line callback registration for one pivot instance
- `register_dash_callbacks_for_instances`: batch callback registration for multi-instance layouts
- `DashPivotInstanceConfig`: typed configuration dataclass for instance wiring
- `DashTanstackPivot` Python component with full prop surface (rowFields, colFields, valConfigs, filters, sorting, expanded, viewport, columnPinning, columnVisibility, conditionalFormatting, drillEndpoint, and more)
- Hierarchical virtual-scroll backend with expand/collapse tree state
- Column pivot support (cross-tab) with dynamic header generation
- Drill-through modal with REST endpoint integration
- Excel/CSV export via SheetJS with hierarchy-aware indentation
- Field zone UI sidebar: drag-and-drop row/column/value/filter zone management
- Column display UI states: resize, pin, hide/show with localStorage persistence
- Ibis backend abstraction supporting DuckDB, ClickHouse, PostgreSQL, BigQuery, Snowflake, and more
- Arrow-native zero-copy data transfer between backend and Dash component
- Redis-optional caching layer (`pip install pivot-engine[redis]`)
- `load_data_from_arrow`: register PyArrow tables directly into the in-memory DuckDB backend
- Comprehensive test suite: session gate, runtime service, Dash callback registration, multi-instance isolation, packaging smoke

### Changed

- README rewritten as consumer-facing documentation with 10-line quickstart and full props reference
- MANIFEST.in simplified: source map declaration removed (not generated by current webpack config)
- `pyproject.toml` placed under `dash_tanstack_pivot/` as the Python distribution root
- `package.json` `prepublishOnly` and `validate-init` hooks removed

### Fixed

- Grand total deduplication generalized via `pivot_spec.rows[0]` instead of hard-coded field name
- Aggregate-aware total computation preserves avg/count/min/max semantics
- Weighted AVG visual totals use hidden count helpers for mathematical correctness
- DuckDB materialized hierarchy creation runs inline to avoid closed pending-query result errors
- `FilterPopover` renders null until anchor geometry exists (prevents rendering at viewport origin)
- Pinned column separators use deterministic `getIsLastColumn`/`getIsFirstColumn` edge detection
- `useColumnDefs` call placed after `useServerSideRowModel` to ensure `renderedOffset` is in scope
